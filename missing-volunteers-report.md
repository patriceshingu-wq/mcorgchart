## Root Cause

The org chart uses a 3-level embedding system:
1. **Ministry** → dark card with embedded department rows
2. **Department** → row inside ministry card, with expandable sub-department rows
3. **Sub-department** → indented row under department, with a member count badge

The problem is at level 3: **individual volunteers under sub-departments are embedded but never rendered**. They only contribute to a count badge.

---

## Recommendations

### Option A: Expandable volunteer lists (recommended)

Add a toggle on each sub-department row that expands to show the volunteer names inline, similar to how departments expand to show sub-departments. This keeps the compact view by default but lets users drill down.

**Pros:** Consistent with existing expand/collapse pattern, keeps UI clean
**Cons:** Adds another nesting level, ministry cards could get very tall

### Option B: Click-to-view detail panel

When clicking a sub-department row (e.g., "Ushers (10)"), open the existing side panel or modal showing all members of that team. The count badge already hints that there are people inside.

**Pros:** Keeps cards compact, reuses existing detail panel
**Cons:** Requires extra click to see names

### Option C: Break out large teams as standalone cards

For sub-departments with more than ~5 volunteers (MC Music English, MC Music French, Ushers, Kids Ministry), render them as standalone white cards connected to the ministry card instead of embedding them. Only embed small teams (Security, Parking, Transportation).

**Pros:** Makes large teams more visible
**Cons:** Increases chart size, loses the compact ministry view for some teams

### Option D: Show all names inline (simplest)

Remove the count-badge-only behavior and always render volunteer names as additional indented rows under each sub-department, like:

```
  Ushers — Binti Tady
    › Jocy-Eunice Ngoy
    › Antoinette Ngonou
    › Audrey Ipoli
    ...
```

**Pros:** All 79 hidden volunteers become immediately visible
**Cons:** Ministry cards become very tall (Worship Experience would have 60+ rows)
